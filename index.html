Lifecycle of Components

Each component in React has a lifecycle which you can monitor and manipulate during its three main phases.

    The three phases are:

  • Mounting

  • Updating

  • Unmounting

Mounting means putting elements into the DOM

React has four built-in Mounting-phase methods:


for initializing local state & binding event handler used constructor() method getDerivedStateFromProps() exists for rare use cases where the state depends on changes in props over time
When called render(), it should examine this.props and this.state and return React Elements
componentDidMount() call in component immediately after mounting. It's a good place to create API queries

Updating

    The next phase in the lifecycle is when a component is updated
    Component can be updated whenever there is a

  • state

  • or props

    • change in the component's

And here is five built-in methods that gets called, in this order, when a component is updated:


getDerivedStateFromProps()
shouldComponentUpdate()
render()
getSnapshotBeforeUpdate()
componentDidUpdate()

The most popular method here it's componentDidUpdate()

Used to implement the functionality based on comparing previous and current props


                                            componentDidUpdate(prevProps) {
                                                if (this.props.ID !== prevProps.ID) {
                                                  this.fetchData(this.props.ID);
                                                }
                                              }

Unmounting

    The next phase in the lifecycle is when a component is removed from the DOM, or unmounting as React likes to call it

  • This phase has only one method:

  • componentWillUnmount()

    • This method performs the necessary reset: cancel all timers, network requests and subscriptions created in componentDidMount()

Angular

Released in 2010 by Google

While React just was in the invention phase, Angular has already released the first version

This framework of Javascript uses real DOM.

It is extremely hard to handle because if you lose the flow, you will have to go deep into the codes to actually find out the issue. It is time-consuming, and risky to attract a lot of bugs.

The real DOM use by Angular affects its performance and ability to make dynamic software application.

Furthermore, it results in the slow performance of this Javascript framework.

But Angular rather a platform than a framework

Because it has so many features built-in like Routing or state management


And at the same time React needs support from other libraries for this tasks

& What about Vue

It is the smallest of other frameworks and libraries. Hence it's extremely suitable for the development of lightweight applications.

Unlike Angular or React, Vue is not backed by any top-level corporate.



This should have resulted in lesser popularity but in reality, this Javascript framework gained amazing popularity in the open source community.

Conclusion

Vue and React offers better performance and flexibility than Angular. Vue and react are more suited for light-weight applications and Angular is the best for large UI applications. Angular is highly opinionated and unlike Vue and React, it offers everything from routing, templates to testing utilities in its package.

So in most cases the choice of frameworks depends on the complexity and size of the application
